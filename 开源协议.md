## 协议说明

本项目采用 **Apache License 2.0** 开源协议，这是一种功能强大且广泛使用的开源协议。

### 您可以做什么？

✅ **商业使用** - 您可以将此软件用于商业目的
✅ **修改** - 您可以修改此软件
✅ **分发** - 您可以分发此软件的副本
✅ **私人使用** - 您可以私人使用此软件
✅ **再许可** - 您可以在修改后使用不同的许可证
✅ **专利使用** - 协议包含明确的专利授权

### 您需要做什么？

📋 **包含版权声明** - 在所有副本或重要部分中包含原始版权声明
📋 **包含许可证** - 在所有副本或重要部分中包含此许可证文本
📋 **说明修改** - 如果您修改了文件，需要说明您的修改
📋 **包含NOTICE文件** - 如果项目中包含NOTICE文件，需要在分发时包含

### 您不能做什么？

❌ **使用商标** - 未经许可不能使用贡献者的商标
❌ **追责** - 软件按"原样"提供，不提供任何形式的担保
❌ **专利诉讼** - 如果您基于此软件发起专利诉讼，许可证将自动终止

### Apache 2.0 协议的特点

1. **专利授权** - 明确授予用户专利使用权，这是与MIT协议的主要区别
2. **专利保护** - 如果用户因专利问题起诉贡献者，将自动失去专利授权
3. **修改声明** - 要求说明修改内容，便于追踪变更
4. **NOTICE文件** - 支持包含额外的版权和许可信息
5. **商业友好** - 非常适合商业项目使用
6. **广泛认可** - 是企业级项目最常用的开源协议之一

### Apache 2.0 vs MIT License

| 特性 | Apache 2.0 | MIT License |
|------|------------|-------------|
| 商业使用 | ✅ | ✅ |
| 修改和分发 | ✅ | ✅ |
| 专利授权 | ✅ 明确授予 | ❌ 无明确条款 |
| 专利保护 | ✅ 有反诉条款 | ❌ 无 |
| 修改声明 | ✅ 要求 | ❌ 不要求 |
| NOTICE文件 | ✅ 支持 | ❌ 不支持 |
| 许可证长度 | 较长 | 简洁 |
| 企业偏好 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

### 为什么选择Apache 2.0？

1. **专利保护** - 提供明确的专利授权和保护机制
2. **企业友好** - 企业更倾向于使用Apache 2.0的项目
3. **法律保障** - 更加详细的法律条款，减少法律风险
4. **广泛使用** - 许多大型项目（如Android、Kubernetes、Spring）都使用此协议
5. **国际认可** - 在国际开源社区中有很高的认可度
